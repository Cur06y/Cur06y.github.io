2.JAVA程序结构

Java的main方法被设计成静态方法主要是为了保证程序的启动独立性和统一入口规范：

  1. 无需对象实例
    在程序启动时，JVM尚未创建任何对象。通过静态方法可以直接调用main，避免了先创建实例才能调用方法的悖论。

  2. 统一入口签名
    固定的public static void main(String[] args)签名确保JVM能准确定位和执行入口方法。

  3. 避免继承冲突
    如果使用实例方法，当多个类包含main方法时，JVM无法确定从哪个实例方法开始执行。

代码结构解析：

  1. 类结构：程序由多个class组成，public类需与文件名一致
  2. main方法：固定格式public static void main(String[] args)
  3. 代码块：使用大括号{}划分作用域
  4. 注释方式：单行// 或多行/*...*/
  5. 命名规范：
    • 类名：大驼峰（Main, Person）
    • 变量/方法：小驼峰（addNumbers, displayInfo）
    • 常量：全大写（MAX_SIZE）

执行流程：
编辑代码 → javac Main.java → java Main → 输出结果
